# Motivation

這次演講主要是在介紹語者辨識系統，以及系統面對冒充者和欺騙攻擊時，可以如何進行防護。
語者辨識是透過說話者的聲音特徵來判斷身分，常見應用像是語音解鎖、電話客服驗證或語音助理等。  
不過，現在的語音合成和語音轉換技術越來越進步，因此單純依靠聲音辨識身分，可能會有安全上的風險。

---

# Speaker Recognition

語者辨識主要可以分成兩種類型：
- 語者驗證
- 語者識別
語者驗證是判斷這個人是不是他宣稱的那個人，例如使用者登入時，系統確認他是否為本人。

語者識別則是判斷這個聲音是誰，系統會從資料庫中的多位語者中找出最接近的人。

---

# Spoofing Attacks

語者辨識系統可能會受到欺騙攻擊。  
攻擊者不一定要真的模仿得很像，也可能使用技術產生假的聲音來騙過系統。
常見攻擊方式包括：
- 人聲模仿（Impersonation）
- 錄音回放（Replay）
- 語音合成（Speech Synthesis）
- 語音轉換（Voice Conversion）

其中錄音回放比較像是實體攻擊，攻擊者透過喇叭播放錄音。  
語音合成和語音轉換則比較偏向數位攻擊，可能直接產生假的語音檔輸入系統。

---

# Text Dependent and Text Independent

語者辨識也可以依照語音內容分成兩種方式。

本文相關 Text dependent 是限制使用者說固定內容，例如指定句子或密碼。  
這種方式通常辨識效果比較好，因為系統可以同時確認聲音和內容。

本文無關Text independent 則是不限制說話內容。  
這種方式使用上比較方便，但辨識難度也比較高，因為系統只能依靠聲音特徵來判斷身分。

---

# System Architecture

這個系統大致可以分成兩個部分。

第一個部分是欺騙攻擊偵測，用來判斷輸入語音是否可能是假語音。  
第二個部分是語者辨識，用來判斷輸入語音是否屬於目標語者。

系統會先從語音中擷取特徵，再丟入不同模型進行判斷，最後整合結果來決定是否通過。

---

# Speech Features

演講中提到幾種常用的語音特徵：
1. MFCC
2. CQCC
3. i-vector
MFCC 是語音辨識中很常見的特徵，可以描述語音的頻譜資訊。
CQCC 是由 CQT 轉換而來，適合用來偵測語音合成或語音轉換留下的不自然痕跡。
i-vector 則可以把不同長度的語音轉成固定長度的向量，方便後續分類與比對。

---

# CQT and CQCC

CQT 的特色是頻率軸採用對數尺度，而不是一般 FFT 的線性尺度。  
因此它在低頻和高頻會有不同的解析方式，比較接近人耳對聲音的感知。
CQCC 則是在 CQT 的基礎上再進一步處理，流程大致為：
1. 輸入語音
2. 前處理
3. CQT 轉換
4. 取對數
5. 均勻採樣
6. 離散餘弦轉換
7. 得到 CQCC 特徵
在這個系統中，CQCC 主要用在欺騙攻擊偵測。

# i-vector

i-vector 是語者辨識中常見的特徵方法。
它的重點是可以把一段長度不固定的語音，轉換成固定長度的特徵向量。  
簡報中提到，系統會將語音轉成約 400 維的 i-vector。

i-vector 的流程大致包括：
1. 擷取 MFCC 特徵
2. 使用 UBM 建立背景模型
3. 取得統計特徵
4. 轉換成固定長度的 i-vector
這樣可以讓不同長度的語音都能用同樣的方式進行比較。

# Ensemble Learning

演講中也提到整體學習，也就是把多個分類器的結果整合起來。
一般常見方法是多數決，只要多數模型認為通過，就讓語音通過。  
但這個系統使用比較嚴格的全數決，也就是所有模型都判斷通過，才會接受。
這樣可以提高安全性，降低冒充者或假語音通過的機率。

---

# Score Normalization

分數正規化是用來調整模型輸出分數的方法。
因為不同語者或不同模型的分數分布可能不同，如果直接比較，可能會不公平。  
透過分數正規化，可以讓判斷門檻比較一致。
不過這種方法也有缺點，因為目標語者和冒充者的分數分布可能仍然會重疊，所以不一定能完全解決誤判問題。

# Training Data and Model

系統訓練資料包含目標語者與冒充者資料。
欺騙攻擊偵測模型使用 CQCC 作為輸入特徵，並使用 DNN 判斷語音是真實語音還是假語音。
模型設定大致包括：

- 輸入大小：60 × 150 = 9000
- 輸出：Real 的機率
- Optimizer：Adam
- Learning rate：0.001
- Loss function：Binary Cross-Entropy
- Batch size：64

# Experimental Results

實驗中有比較不同降維方式的效果，例如使用 PCA 或 LDA 降低特徵維度。
從結果來看，原始特徵的 Real 通過率最高，但 Spoofing 通過率也仍然存在。  
使用 PCA 降維後，模型大小和計算量可以降低，但辨識效果可能會受到影響。
因此在設計系統時，需要在準確率、模型大小和安全性之間取得平衡。

# Conditional Retry
為了避免合法使用者被錯誤拒絕，系統可以加入條件式重試機制。
也就是當系統判斷結果不明確時，讓使用者重新錄音 1 到 2 次。  
這樣可以降低正常使用者被拒絕的機率。
但重試次數不能太多，否則可能會被攻擊者利用。
