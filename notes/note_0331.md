### 題目: 如何打造台版 Edge Impulse TinyML 開發平台 Part 2

#### 學號: 11463111
#### 姓名: 劉柏毅

今天的主題是研究Edge AI以及TinyML是什麼以及和生成式 AI 有什麼不同，需要的硬體與工具，開發流程以及台灣做自己的Edge Impulse平台，應具備哪些功能
邊緣智慧的特性有:
1. 邊緣端資源很少: 儲存（Flash / RAM）有限，計算能力有限
2. 智慧感測: 聲音，影像，運動
3. 超小模型，超低功耗
目標是：極小化資源消耗
生成智慧特性:
1. 雲端無限資源: 儲存，計算，頻寬，功耗
2. AI生成: 對話，影像，程式
3. 超巨大資料集，模型
目標是：能力極大化

AI 發展趨勢
感知 AI->生成 AI->代理 AI->物理 AI
推動因素包含：
1. AI 核心技術
2. 半導體技術
3. 計算能力
4. 開發工具
5. 基礎模型
6. 多模態技術

不同公司對Edge AI的範圍理解不同
1. Edge Server
2. NB AI（AI PC）
3. Mobile AI（SoC）
4. SBC AI（MPU / NPU / GPU）
5. Wearable AI（MCU / TinyML）
   
Edge AI定義
廣義:不上網就可以完成所有AI推論
狹義:使用電池供電完成所有AI推論的獨立裝置

Edge AI 開發流程
資料收集->資料標註->模型建置->訓練調參->模型優化->佈署測試

為什麼需要台版 Edge Impulse
1. 提升嵌入式人工智慧工程便利性
2. 加速 Edge AI 與 TinyML 開發效率
3. 提供國內產官學研整合平台
4. 共享資源，快速提升工程能力
5. 高度整合國內晶片與軟體業者
功能配置:
1. 雲端及本機虛擬 / 實機開發模式
2. 資料資產管理工具
3. 模型訓練 / 調參工具
4. 模型優化 / 部署工具
5. MLOps 管理

#### 心得
這場演講讓我了解，Edge AI和TinyML不只是把AI放進小裝置裡，是在資源有限且低功耗的條件下完成即時推論，和生成式AI的方向不太一樣，這次演講也讓我知道Edge AI的重點不是放在模型本身，還包含晶片，開發工具，模型優化和實際部署的流程，也發現Edge AI是一個跨多個領域的應用，有AI模型和嵌入式系統以及晶片架構與軟體工具，雖然模型規模和算力不像生成式AI那麼大，但即時性，低功耗以及本地運算上有自己的優勢，這場演講也讓我對EDGE AI和TinyML以及如果台灣要建立自己的Edge Impulse平台需要哪些功能，有了更多的了解。
